# MXOS 1.7

> 一个运行在浏览器中的桌面操作系统体验项目。

MXOS 以单页 Web 应用的形式模拟完整的桌面环境：桌面、任务栏、窗口管理、开始菜单、设置和一组内置应用均可直接在现代浏览器中运行。项目无需 Node.js 构建流程；启动本地静态服务器后即可使用。

## 特性

- **桌面体验**：壁纸、桌面图标、任务栏、通知、开始菜单、右键菜单与锁屏界面。
- **窗口管理**：拖动、缩放、最小化、最大化、贴靠、分屏、虚拟桌面及多种窗口效果。
- **内置应用**：浏览器、记事本、计算器、日历、时钟、音乐、Office、文件管理器、终端、应用商店、绘图与像素编辑器等。
- **个性化设置**：明暗主题、强调色、字体缩放、壁纸颜色与轮播、动画等级、音效和光标效果。
- **效率功能**：启动搜索、命令面板、剪贴板历史、便签、倒计时、小组件、专注模式与使用报告。
- **设备与媒体能力**：屏幕录制、二维码、OCR、媒体设备、传感器、电池和性能信息（由浏览器权限与设备支持情况决定）。
- **沉浸式彩蛋**：环境音、桌面宠物、粒子艺术、星空、节日彩蛋、键盘音乐等互动功能。
- **本地优先**：应用状态和虚拟文件系统在浏览器端运行；部分联网、位置或设备功能需要相应权限。

## 快速开始

### 方式一：使用项目自带服务器（推荐）

要求：Python 3。

```bash
cd F:\MXOS\1.7
python serve.py
```

服务器会启动在 `http://localhost:8080`，并尝试自动打开浏览器。使用 `Ctrl+C` 停止服务。

### 方式二：使用 Python 内置静态服务器

```bash
cd F:\MXOS\1.7
python -m http.server 8080
```

然后访问 `http://localhost:8080`。

> 请不要直接以 `file://` 协议打开 `index.html`。Service Worker、浏览器权限和部分 Web API 在 HTTP 本地服务器下运行更可靠。

## 项目结构

```text
.
├── index.html              # 应用入口、页面结构与主要样式
├── sw.js                   # Service Worker / 离线缓存逻辑
├── serve.py                # 禁用缓存的本地开发服务器
├── MiSans-Normal.ttf       # 本地字体
├── _redirects              # 静态站点重定向配置
└── js/
    ├── main.js             # 应用启动入口
    ├── core.js             # 桌面与窗口核心能力
    ├── state.js            # 状态管理
    ├── vfs.js              # 浏览器端虚拟文件系统
    ├── apps/               # 内置应用
    ├── features/           # 系统特性与互动效果
    ├── settings.js         # 设置界面与配置
    ├── i18n/               # 中文 / 英文本地化资源
    ├── media/              # OCR、二维码与录屏功能
    ├── real/               # 设备、性能和传感器接口
    ├── system/             # 备份、恢复、更新与诊断功能
    └── utils/              # 通用工具模块
```

## 使用说明

- 从桌面图标、开始菜单或搜索入口启动应用。
- 大多数窗口支持拖动标题栏、边缘缩放，以及最小化、最大化和关闭操作。
- 可在**设置**中调整主题、壁纸、动效、声音、字体与其他个性化选项。
- 需要摄像头、麦克风、位置、传感器或屏幕共享的功能，会在浏览器中请求权限；拒绝权限后相应功能不可用。
- 项目通过 CDN 加载 JSZip，用于部分压缩与导入导出能力；离线使用时该依赖可能受网络缓存状态影响。

## 浏览器兼容性

推荐使用最新版 Chrome、Edge 或其他 Chromium 浏览器。部分功能依赖以下现代 Web 平台能力：

- Service Worker 与 Cache API
- IndexedDB / LocalStorage
- MediaDevices、Screen Capture 与 Permissions API
- Web Speech、Geolocation、Device Orientation 等可选 API

不同浏览器、设备和安全策略下，可用功能会有所差异。

## 部署

这是一个静态 Web 应用，可部署到任意支持静态文件托管的服务。部署时请确保：

1. `index.html`、`js/`、`sw.js` 和 `MiSans-Normal.ttf` 保持相对路径关系；
2. 使用 HTTPS（本地 `localhost` 除外），以保证媒体、位置和 Service Worker 等功能正常工作；
3. 站点根目录部署时保留 `_redirects`；若部署在子路径，请相应检查 Service Worker 的作用域与资源路径。

## 开发

本项目当前不依赖打包器或前端框架。修改 JavaScript 模块或 `index.html` 后，刷新页面即可验证；`serve.py` 已设置无缓存响应头，便于开发时获取最新文件。
